## 2. 연구 동기 — 블랙박스 문제

### 2-1. 문제 정의

현대 AI(특히 ANN 기반 시스템)는 **블랙박스 문제**를 내재한다. 입력을 결과로 변환하는 내부 계산 과정을 인간이 추적할 수 없다는 구조적 한계다. Anthropic社는 이를 다음과 같이 표현했다:

> *"A surprising fact about modern large language models is that nobody really knows how they work internally."*

이 문제는 의료·금융·국가 행정·군사 등 안전성이 1순위인 분야에서 AI 도입을 근본적으로 제약한다.

### 2-2. 블랙박스 문제의 구조적 원인

블랙박스 문제는 단순한 복잡도의 문제가 아니라 **ANN의 설계 구조** 자체에서 발생한다:

1. **가중치의 의미 부재**: 학습된 가중치는 정답률 최적화의 결과일 뿐, 그 자체로 의미(기의)를 갖지 않는다. 각 가중치는 단순한 숫자(기표)이며, 강아지의 '귀'라는 개념을 직접 인코딩하지 않는다.

2. **분산 표상(Distributed Representation)**: '중요한 특징'은 단일 뉴런이 아니라 수많은 뉴런에 걸쳐 분산되어 처리된다. 강아지-고양이 분류에서 '귀'라는 개념은 수천 개 뉴런으로 미시화된다.

3. **중첩(Superposition)**: Anthropic 연구(Elhage et al., Bricken et al.)에 따르면, 단일 뉴런이 서로 무관한 여러 개념(학술 인용문, 영어 대화체, HTTP 요청, 한국어 텍스트)을 동시에 인코딩한다. 따라서 '중요한 뉴런'을 찾더라도 그 반응 원인을 알 수 없다.

4. **Transformer의 공유 계산 공간**: 현대 LLM의 수천 개 층이 하나의 공유 계산 공간에 결과를 겹쳐 쓴다. 레이어별 데이터 흐름 추적이 사실상 불가능해진다. Anthropic 자사 모델에 완전 접근 가능한 조건에서도 계산 경로의 절반만 재구성 가능했다.

## 3. 연구 목적 — 사전 해석가능성(Prior Interpretability)

### 3-1. 핵심 아이디어

**인공신경망 대신 동역학적 과정을 계산의 본위**로 삼는 새로운 분류 시스템을 설계·구현한다.

- 데이터를 **물리적 객체(입자 앙상블)**로 취급
- 입자에 초기 조건을 부여하고 **동역학적 흐름**으로 데이터를 처리
- 데이터 처리의 전 과정이 **운동방정식으로 완전히 기술**됨
- 수식화된 과정은 언제든 해체·분석 가능

### 3-2. 기존 ANN과의 근본적 차이

> **단순한 ANN들은 해석이 가능하나, 그것은 복잡도가 낮아서 해석이 가능한 것이지, 구조적으로 해석 가능하게 설계된 것이 아니다.**

본 시스템은 문제의 복잡성과 무관하게 **물리 방정식이 계산 과정을 고정**하므로, 얼마나 복잡한 문제를 다루든 추론 과정의 수식적 추적이 구조적으로 보장된다.

해밀턴 역학에서는 두 개의 일계상미분 방정식이, 라그랑주 역학에서는 오일러-라그랑주 방정식이 이를 보장한다.

### 3-3. ANN 요소 치환 대응표

| ANN 요소 | CHM 대응 요소 |
|----------|--------------|
| 가중치 행렬 (weight matrix) | 포텐셜 장 V(x) |
| 데이터 벡터 | 물리적 입자의 집합 (앙상블) |
| 행렬 연산 (순전파) | 접촉 해밀토니안 동역학 (ODE 적분) |
| 학습 | 에너지 지형 V(x) 자체의 최적화 |

### 3-4. "V(x)도 숫자인데 어떻게 해석하는가?" 반론에 대한 답변

신청서 원문에서 명시적으로 다룬 반론:

> *'가중치가 기표일 뿐 기의가 없기 때문에 해석할 수 없다면, 똑같이 숫자일 뿐인 V(x)는 어떻게 해석하는 것인가?'*

**답변**: 본 연구가 추구하는 해석 방향은 **V(x) 파라미터 자체가 아니라 그 위에서 전개되는 완전히 추적 가능한 운동**이다.
- V(x)는 물리적 의미(에너지 지형의 기의)를 가진다
- 더 근본적으로는, V(x)가 어떤 값이든 **그 위에서의 계산 과정은 운동방정식으로 완전히 기술**된다
- 시야를 "파라미터 자체의 해석"에서 "파라미터 위에서 전개되는 추론 과정의 추적"으로 이동

## 4. 시스템 개요 — Contact Hamiltonian Machine (CHM)

### 4-1. O/X 이진분류 구체 구현

입력 이미지(8×8 픽셀)의 각 픽셀을 입자로 치환하여:

1. **포텐셜 장** V(x)가 만드는 에너지 지형(산과 골짜기) 위에 배치
2. **소산계수 γ**에 의해 에너지를 잃으며 골짜기 최저점으로 이동
3. O-클래스용 골짜기 → `q*_O = (8, 8, 0)`, X-클래스용 골짜기 → `q*_X = (-8, -8, 0)`
4. 입자 앙상블의 무게중심(CoM)이 어느 골짜기로 수렴하는지를 분류 결과로 판정

**학습** = 이 에너지 지형 V(x) 자체를 경사하강으로 최적화하는 과정

### 4-2. 핵심 물리 방정식 (접촉 해밀토니안)

```
dq/dt  =  p
dp/dt  =  -∂V/∂q  -  γ p
dz/dt  =  ||p||²  -  H         (H = ||p||²/2 + V(q))

=> dH/dt = -γ ||p||²  ≤  0    (에너지 단조 감소, 수렴 보장)
```

V(x)의 구체적 형태: K=16개의 RBF(방사 기저 함수)의 가중합

```
V(q) = Σₖ wₖ · exp(-||q - μₖ||² / (2σₖ²))
```

학습 파라미터 θ = {wₖ, μₖ, σₖ}: k=0,1은 frozen(attractor 고정), k=2..15는 학습

### 4-3. EBM으로서의 위치

본 시스템은 에너지 기반 모델(EBM)의 일종이나, **에너지를 최소화하는 과정이 알고리즘이 아닌 물리 법칙(접촉 해밀토니안 역학)으로 수행**된다는 점이 차별점이다.

## 6. 코드 규약 (Claude Code 작업 시 필수 숙지)

- **언어/백엔드**: Python 3.10+, JAX (CUDA), optax
- **`rk4_step`**: `static_argnums=(4, 5)` — `gamma`, `dt`는 항상 Python 스칼라(정적)
- **`preprocess()`**: 3D Lifting 사용. `z_i = sigmoid(β·(d_axis - d_diag))` 로직 보존 필수
- **`simulate_diff()`**: `jax.checkpoint` + `lax.scan` BPTT. scan body에서 `rk4_step` 단일 호출
- **σ 파라미터**: `softplus(sigma_raw) + 0.1` 재파라미터화 (minimum 0.1 보장)
- **PyCharm 호환**: `_rbf_potential = rbf_potential` 등 모듈 수준 명시적 참조 사용
- **출력 경로**: `Path(__file__).resolve().parent` 기반 (`_HERE` 변수)
